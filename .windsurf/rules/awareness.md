# Rule: Context First

**Always read and analyze the current open folder structure before making any changes or taking any action.**

## Purpose
Understand the existing environment and context before modifying or adding to it. This prevents conflicts, redundancy, and unintended consequences.

## When to Apply
- Before creating new files or folders
- Before editing existing files
- Before restructuring or moving content
- At the start of any new task or session

## Why It Matters
- Ensures awareness of what already exists
- Reveals patterns and conventions to follow
- Prevents breaking existing relationships between files
- Saves time by avoiding blind decisions
